> **Auto-generated by the [Session Scribe](/docs/session-scribe/).** A robot wrote this from a Claude Code session; a human reviewed the pull request before it shipped. This first one was seeded by hand in the scribe's own format — future dispatches arrive automatically when a session ends.

This session built the thing that writes these. The ask: a Claude Code hook that,
at the end of each session, turns the work into a shareable article so nobody has
to redo it.

**What was attempted.** First instinct was a `Stop` hook (fires every turn) — wrong
tool; it would write an article after every message. The right event is
`SessionEnd`: it fires once when the session closes, can't block teardown, and
hands the hook `session_id`, `transcript_path`, and `reason`. Confirmed against
the Claude Code hooks docs before committing to it.

**What broke, and the fix.** Three real traps surfaced:

1. **Recursion.** A hook that runs `claude` will, by default, load the same
   project hooks and fire `SessionEnd` again — forever. There is no built-in
   "I'm inside a hook" flag, so the fix is a self-managed guard env var
   (`CLAUDE_SESSION_SCRIBE=1`) checked at the top of the script, plus `--bare`
   on the inner call to skip hook discovery entirely. Belt and braces.
2. **A GNU-ism that no-ops on macOS.** Stripping the title line with
   `sed '0,/^# /d'` silently did nothing on BSD `sed`. The test caught it. Swapped
   to `awk` — portable across both.
3. **Secrets.** Transcripts contain whatever you typed, including tokens. The
   writer is told to omit them, the script scrubs known credential patterns, and
   the whole thing lands as a *draft PR* a human reads first. The review gate is
   the real safety, not the regex.

**How it was verified.** An offline test injects a fake writer and a temp output
dir, so it never calls the model: it asserts the recursion guard exits, trivial
sessions are skipped, the front matter is right, secrets are redacted, and a
second run for the same session adds no duplicate. 13/13 green.

## Takeaways

- For "once per session" work, use `SessionEnd`, not `Stop`.
- Any hook that invokes `claude` needs a recursion guard *and* `--bare`.
- Test shell scripts on the OS you'll run them on — `sed 0,/re/` is GNU-only.
- When a robot publishes to the public internet, the human-reviewed PR is the
  safety mechanism. Everything else is defense in depth.
